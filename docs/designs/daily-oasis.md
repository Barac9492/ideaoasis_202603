## Vision

### 10x Check
The 10x version isn't "more sources" — it's a Korean startup intelligence platform. 50K Korean entrepreneurs open IdeaOasis daily. The AI identifies **timing windows** ("This category growing 40% YoY in US, zero Korean players, 6-month window"). It tracks which ideas got built by Korean teams and whether they succeeded. It becomes the definitive record of idea migration across the Pacific.

### Platonic Ideal
The user opens IdeaOasis at 7 AM with coffee. Five beautiful cards. Each makes them think "I could build that." They tap one — rich Korean analysis with Naver search data, specific competitor gaps, regulatory notes. They screenshot it and send to their co-founder via KakaoTalk. The site feels like it was made by someone who deeply understands both Silicon Valley and Gangnam.

## Landscape Context

**Key finding:** IdeaBrowser (Greg Isenberg, May 2025) already does ~80% of this in English — AI-curated startup ideas from Reddit with daily delivery. The Korean market lens (Naver data, Korean competitor tracking, KakaoTalk sharing) is the only defensible differentiator.

**Three-layer synthesis:**
- Layer 1: ProductHunt + Reddit aggregation is tried-and-true
- Layer 2: IdeaBrowser is the current standard; Disquiet.io is the Korean community
- Layer 3: The value isn't translation — it's Korean-specific demand signals (Naver DataLab) that no English platform can provide

## Implementation Approach

**Chosen: Approach B — Start with ProductHunt**
- Single reliable API source (PH GraphQL, 6250 complexity points / 15 min)
- Add Reddit as source #2 after pipeline is stable
- Skip X (expensive API) and IdeaBrowser (no API, fragile scraping) for MVP

**Data storage:** JSON files committed to git repo. Cron generates, commits, Vercel auto-deploys. Zero infrastructure.

## Pipeline Schedule & Error Handling

**Cron:** GitHub Actions at `0 20 * * *` UTC (= 5:00 AM KST), runs 7 days/week. Pipeline completes by ~5:30 AM KST. Vercel build takes 1-3 min after push, so content is live by ~5:35 AM KST — well before the 7 AM target.

**Why GitHub Actions over Vercel Cron:** GitHub Actions allows longer execution (up to 6 hours), can run `git commit` + `git push`, and is free for public repos. Vercel Cron on free tier is limited to daily and 10-second execution.

**Error handling:**
- PH API down → Retry 2x with 30s backoff. If still failing, skip today's digest and retain previous day's content on the homepage. Log error to GitHub Actions output.
- Claude API error → Same retry logic. If Claude fails, skip AI analysis and publish raw PH data with a note "AI 분석 준비중" (AI analysis preparing).
- Naver DataLab failure → Non-blocking. Publish idea without trend data. Show "트렌드 데이터 없음" (No trend data) in the UI.
- The homepage always shows a "마지막 업데이트" (Last updated) timestamp so users know if content is stale.

## Naver DataLab Integration Spec

**API endpoint:** Naver DataLab Search Trend API (`https://openapi.naver.com/v1/datalab/search`). (Shopping Insight API is for e-commerce categories — not relevant here.)

**Keyword extraction:** Claude generates 1-2 Korean search keywords per idea as part of the analysis step. These keywords are then queried against Naver DataLab Search Trend API.

**API constraints:** Naver DataLab Search Trend API accepts up to 5 keyword groups per request, each with up to 20 keywords. Returns a relative index (0-100) over a specified date range. Free tier allows 1000 calls/day — more than sufficient for 5 ideas.

**Aggregation rule:** `trend_index` = most recent monthly value from the Naver response. `trend_direction` = "up" if current month > 3-month average, "down" if lower, "flat" otherwise.

**UI display:** Each idea card shows a small trend indicator — up arrow (green), flat (gray), or down arrow (red) — with the trend index number and period. The idea detail page shows a mini line chart of the trend over the past 12 months.

## KakaoTalk Share Spec

**MVP approach:** Use Kakao Link API basic URL share (`Kakao.Link.sendDefault`) which does NOT require template review. This sends a standard link message with title, description, and thumbnail — auto-populated from the page's OG tags.

**Why not custom template:** Custom message templates require Kakao app review (days-weeks). The basic URL share is instant and still generates a rich preview card from OG metadata. Upgrade to custom template post-launch.

## OG Image Spec

Pre-generated at build time using Satori (the library behind `@vercel/og`) directly as a build script — NOT via Edge Route (incompatible with `output: 'export'`). The build script reads each idea from JSON data, generates `.png` files, and uploads them to **Cloudinary** (free tier: 25GB). OG meta tags reference Cloudinary URLs. This avoids bloating the git repo (~100MB/year of PNGs). Each idea detail page gets a unique OG image:
- 1200x630px card
- IdeaOasis logo (top-left)
- Idea title in Korean (Pretendard Bold, large)
- English tagline below (smaller, gray)
- Naver trend indicator (if available)
- Daily rank badge (#1 through #5)
- Gradient background matching the site's color scheme

Daily digest page (`/`) gets a generic OG image: "오늘의 스타트업 아이디어 TOP 5" with the date.

## Weekly "한 주의 TOP" Spec

Every Saturday at 6:00 AM KST, a second GitHub Actions cron runs:
1. Reads the week's 7 daily digests (Mon-Sun, 35 ideas total)
2. Claude selects the single best idea based on: novelty, Korean market opportunity, and overall upvotes
3. Generates an extended analysis: deeper market sizing, step-by-step Korean localization playbook, comparable Korean success stories, estimated build timeline
4. Writes to `data/weekly/YYYY-WNN.json`
5. Displayed at `/weekly/[date]` with a distinct "weekly deep dive" layout

**Weekly error handling:** If fewer than 3 daily digests exist for the week (pipeline failures), skip the weekly summary and retain the previous week's at `/weekly/latest`. If Claude fails during weekly analysis, retry 2x, then skip with a log.

## Dark Mode Design Notes

Korean aesthetic = clean, generous whitespace, subtle gradients. Inspired by:
- **Toss (토스):** High-contrast cards on dark backgrounds, generous padding, clear hierarchy
- **Baemin (배민):** Playful typography, rounded corners, warm accent colors

Concrete: Tailwind dark mode via `class` strategy. CSS custom properties for theme colors. Light = warm white (#FAFAF9) background. Dark = deep charcoal (#1A1A1A). Accent: warm coral (#FF6B6B) as default — can be refined in `/design-consultation`. Pretendard font self-hosted for performance.

**Why dark mode is MVP scope:** The Korean design aesthetic (Toss/Baemin-inspired) IS the product's credibility signal. A site that looks like a generic English template won't earn trust with Korean entrepreneurs. Dark mode + polished Korean typography is table stakes, not polish.

## Required API Keys (all stored as GitHub Actions secrets)
1. `PH_API_TOKEN` — ProductHunt API (register app → OAuth token)
2. `ANTHROPIC_API_KEY` — Claude API (Anthropic)
3. `NAVER_CLIENT_ID` + `NAVER_CLIENT_SECRET` — Naver DataLab API (Naver Developers account)
4. `KAKAO_JS_KEY` — Kakao JavaScript SDK (Kakao Developers registration — no app review needed for basic URL share). Also set as `NEXT_PUBLIC_KAKAO_JS_KEY` for client-side use.

## Pages
- `/` — Today's digest (top 5 ideas with cards, last-updated timestamp)
- `/archive` — Past digests by date (20 per page, URL: `/archive/[page]`, all pages SSG at build time)
- `/idea/[id]` — Individual idea with full Korean analysis + Naver trend chart (`id` = `ph-{source_id}`)
- `/weekly/[date]` — Weekly best-of deep dive with extended analysis
- `/about` — Static page: project description, data sources, creator info
- `/feed.xml` — RSS/Atom feed (last 30 daily digests, each digest = one `<entry>` with title "IdeaOasis {date} — 오늘의 TOP 5", content = 5 idea summaries with Korean titles and one-line descriptions, link to `/` for that day)

## Build Performance Note

Use Next.js `output: 'export'` for full static export. As the archive grows, use dynamic imports and code splitting to keep builds fast. At 365 digests/year with 5 ideas each, the data is ~365KB of JSON — trivial for build performance. Revisit if/when archive exceeds 1000 digests.
